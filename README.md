Real-Time HIL Sensor Validation Platform
Overview

This project is a real-time Hardware-in-the-Loop (HIL) inspired sensor validation platform developed in Python.
The system simulates live sensor telemetry, processes the data through ECU decision logic, validates system behavior, stores execution history, and visualizes the complete pipeline using a real-time dashboard.

The platform demonstrates concepts commonly used in:

Embedded systems validation
Automotive ECU testing
HIL simulation environments
Real-time telemetry monitoring
Fault detection systems

ystem Architecture
Live Sensors
      ↓
ECU Processing Layer
      ↓
Validation Engine
      ↓
Logging System
      ↓
Persistent Telemetry History
      ↓
Real-Time Dashboard

Working Principle

The application runs continuously in a loop and performs the following operations every second:

Generate live sensor values
Simulate telemetry fluctuations
Process sensor inputs through ECU logic
Validate system state
Store execution data in logs
Display live telemetry in dashboard
Sensor Simulation

The sensor layer generates dynamic telemetry values for:

Temperature
Pressure
Vibration

Example:

{
  "temp": 84.2,
  "pressure": 29.7,
  "vibration": 4.3
}

The generated values simulate fluctuating real-world sensor behavior.

ECU Logic

The ECU module processes sensor data and detects abnormal operating conditions.

Decision Rules
Condition	ECU Response
Temperature > 95	OVERHEAT WARNING
Pressure < 28	LOW PRESSURE ALERT
Vibration > 6	MECHANICAL FAULT
Otherwise	NORMAL

The ECU acts as the decision-making layer of the system.

Validation Engine

The validation module classifies ECU responses into system health states.

ECU Result	Validation Status
NORMAL	PASS
Warning/Fault	FAIL

This provides automated system verification.

Logging System

The logging layer stores all execution data in:

logs/live_log.json

Each entry contains:

Timestamp
Sensor telemetry
ECU response
Validation status

Example:

{
  "sensor_data": {
    "temp": 101.4,
    "pressure": 31.8,
    "vibration": 5.0
  },
  "ecu_result": "OVERHEAT WARNING",
  "status": "FAIL",
  "timestamp": "12:01:02"
}

The logging system enables:

execution history tracking
telemetry monitoring
debugging
traceability
Real-Time Dashboard

The dashboard is built using Streamlit and continuously displays:

Live sensor telemetry
ECU outputs
Validation status
Recent execution history

The dashboard automatically refreshes using the latest log entries generated by the live execution engine.
